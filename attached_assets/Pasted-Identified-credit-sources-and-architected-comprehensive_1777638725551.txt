Copy this entire prompt and give it to Claude, GPT-4, or Gemini:

You are a senior full-stack AI engineer. Help me build "AI SkillFit" — 
a mobile-first, voice-driven video interview and workforce assessment 
platform for Karnataka's blue-collar job candidates. This is for a 
7-day hackathon. Here is the full project spec:

---

PROJECT: AI SkillFit
GOAL: A candidate opens a WhatsApp link on their Android phone, 
completes a 5–7 question voice+video interview in Kannada/Hindi/English, 
and gets classified into a workforce category. Government officers see 
all candidates on an admin dashboard.

TECH STACK (all free):
- Frontend: React PWA (no app install, works on 2G Android)
- Backend: FastAPI + PostgreSQL
- Speech-to-Text: Groq Whisper API (free tier)
- LLM Scoring: Gemini 1.5 Flash API (free tier)
- Face/Liveness: OpenCV + MediaPipe (open source)
- Duplicate detection: face_recognition library (dlib embeddings)
- Hosting: Supabase (DB + storage) + Render.com (backend)

---

CANDIDATE FLOW:
1. Candidate receives WhatsApp link
2. Opens PWA in browser — no install
3. Enters name + phone number (OTP optional for prototype)
4. AI agent greets them by voice in auto-detected language 
   (Kannada default, Hindi, English)
5. AI asks 5–7 trade-specific questions via text-to-speech
   - Example trades: Welder, Electrician, Carpenter, Plumber
   - Questions are pre-written per trade, played as audio
6. Candidate records video response for each question 
   (30 sec max per question)
7. On submission:
   - Groq Whisper transcribes audio (handles Kannada dialects, 
     accents, code-mixing)
   - Gemini 1.5 Flash scores each response on:
     * Relevance (0–10): Did they answer the question?
     * Clarity (0–10): Was communication understandable?
     * Confidence (0–10): Did they show domain knowledge?
   - Rubric instruction to Gemini: "Do NOT penalise dialect, 
     informal phrasing, or code-mixing. Score only content."
   - OpenCV checks face presence every 5 seconds of video
   - MediaPipe liveness check (blink/movement detection)
   - face_recognition generates face embedding, checks DB 
     for duplicate faces
   - Phone number + device fingerprint also checked for duplicates
8. Candidate receives classification:
   - Job-Ready: avg score ≥ 7, clean integrity
   - Requires Training: avg score 4–6, clean integrity
   - Manual Verification: borderline scores OR mild integrity flag
   - Poor Quality: audio/video too degraded to score
   - Suspected Duplicate: face/phone/device match found

IMPORTANT: No candidate is ever auto-rejected. All flagged 
candidates go to human officer review.

---

- Login for government officers
- Table of all candidates with: name, trade, district,

- Filters: by district, trade, language, classification
